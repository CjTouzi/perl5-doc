=EN How to Build a Horse
=CN 如何创建 I<Horse>类
=EN If we constructed all our horses by hand, we'd most likely make mistakes from time to time.
=CN 如果我们手工创建所有的马, 那我们很可能一次次犯错。
=EN Making the "inside guts" of a Horse visible also violates one of the principles of OOP.
=CN 而且暴露 I<Horse> 内部的信息也违反了面向对象编程的基本原则。
=EN That's good if we're a veterinarian but not if we just like to own horses.
=CN 我们不是兽医, 我们只想拥有一匹马而已。

=EN Now, with the new named method, we build a Horse:
=CN 现在, 用新的 I<named> 方法, 我们来创建一个 I<Horse:>
=EN We're back to a class method, so the two arguments to Horse::named are "Horse" and "Mr. Ed".
=CN 参考类方法, 所以有两个参数传给 I<Horse::named> , I<"Horse"> 和 I<"Mr.Ed".>
=EN The bless operator not only blesses $name, it also returns the reference to $name, so that's fine as a return value.
=CN I<Bless> 操作符不仅"祝福"了 I<$name> , 也返回了 I<$name> 的引用, 所以返回值是对的。
=EN And that's how we build a horse.
=CN 我们就是这样创建一个 I<horse> 对象。
=EN We called the constructor named here so it quickly denotes the constructor's argument as the name for this particular Horse.
=CN 我们调用构建函数 I<named> 来快速地把参数作为马的名字。
=EN We can use different constructors with different names for different ways of "giving birth" to the object (such as recording its pedigree or date of birth).
=CN 我们可以用不同名字的构建函数来给对象加上生日（如谱系记录或生日）。
=EN However, we'll find that most people use a single constructor named new, with various ways of interpreting the arguments to new. Either style is fine, as long as we document our particular way of giving birth to an object.
=CN 然而, 大多数人喜欢用 I<new> 来命名构建函数, 并对不同的参数以不同的解释.只要能太到目的, 两种方法都可以。
=EN Most core and CPAN modules use new, with notable exceptions, such as DBI's DBI->connect( ). It's really up to the author.
=CN I<CPAN> 上大多数模块用 I<new> , 也有些是例外, 比如 I<DBI> 模块的 I<DBI->connect().> 这根据开发者的习俗。
